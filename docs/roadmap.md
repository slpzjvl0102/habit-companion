# Roadmap

## v1 — NOW: the experiment (~3 weeks)
제품이 아니라 **반증 가능한 N=1 실험.** 단 하나의 질문에 답하려는 것:
**"컴패니언 *돌봄* 루프가 진짜 습관을 만드나?"** ( = 컴패니언을 치우면 5분 독서가 *유지되나?* )

### v1에서 검증하는 것
- **PRIMARY (행동):** 5분 독서가 컴패니언 OFF 구간(A2)에서도 *시키지 않아도* 유지되는가. ← 이게 핵심.
- **보조(받침대):** 10살이 노벨티 지나서도 컴패니언에 붙어 있나(가정 B)? 관리자가 *잔소리 없이* 도 되나? 관리자가 "의지가 아니라 환경이 했다"를 체감하나(정성 메모, 성공 게이트는 아님).
- **검증 안 함(범위 밖):** 수요(다른 가정이 원하나), 사업성, N=1 너머 일반화.

### ABA 설계 (단일사례 반전설계)
- **A1 베이스라인 (3~5일):** 앱/컴패니언 없이 현재 빈도만 관찰 (docs/baseline-sheet.md).
- **B 개입 (2주):** 컴패니언 ON (앱).
- **A2 반전 (4~5일):** 컴패니언 OFF, 같은 시트로 계속 관찰.
- **읽는 법:** B에서 오르고 **A2에서도 유지** → 신호(뭔가 남았다). B에서 올랐다 **A2에서 붕괴** → 프롭이었다(습관 아님). A1에서 이미 높음 → 원래 습관(그래서 '실패 여지 있는' 5분 독서를 고름).
- **주의(정직):** N=1 ABA는 *방향 신호*지 증명 아님(대조군 없음, 1명, 교란 잔존). 3주 < 습관 정착 ~2개월(Lally) → "초기 신호"로 읽기.

## 결정 게이트 (A2 후)
- **신호 있음** → v2 투자.
- **신호 없음** → 돌봄 메커니즘은 습관을 못 만든 것. 더 쌓지 말고 *방향 전환* (예: founder-fit 있는 싱글/어른 모드). 몇 달 아낌.

## v2 — 실험이 신호를 보이면
- **AI 말하는 컴패니언** [v1 빌드 세션에서 로그]: 펫이 성격 있는 격려 대사 생성·반응(예: Claude API). 단 LLM 키는 클라이언트(PWA)에 못 넣음 → **작은 백엔드(서버리스) 필요.**
- 원격/동기화 백엔드: 관리자가 *자기 폰* 에서 승인, 다기기 공유.
- 네이티브 APK(Android Studio) + 튼튼한 푸시 알림(저녁 리마인더).
- 컴패니언 깊이: 이름짓기·정체성, 테마 선택, 풍부한 진화; '고리' 시각 메타포.
- 습관 템플릿 라이브러리(나이·목표별); 실물 보상 자동화(용돈).
- 가능하면: 싱글/어른 모드(founder-fit), 피험자 확대.
